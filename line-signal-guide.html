คู่มือระบบส่งสัญญาณเข้า LINE

อธิบายตั้งแต่การตั้งค่า Admin โฟลว์การส่งสัญญาณ ไปจนถึงความหมายของข้อมูล Entry / SL / TP ที่สมาชิกเห็นในกลุ่ม LINE

ภาพรวมระบบ

เมื่อ EA บน MT5 ตรวจพบสัญญาณ BOS หรือ CHoCH จาก LuxAlgo SMC ระบบจะส่งข้อมูลมายัง Backend ของ ATH Trader จากนั้นระบบจะ:

  1. บันทึกสัญญาณลงฐานข้อมูล
  2. เรียก AI วิเคราะห์ (หากเปิดใช้)
  3. ตรวจสอบความมั่นใจและโหมดอนุมัติ
  4. ส่งข้อความแจ้งเตือนไปยังกลุ่ม LINE เป้าหมายตามที่ตั้งไว้
ข้อควรรู้

ระบบรองบับการส่งไปหลายกลุ่มพร้อมกัน โดยแต่ละกลุ่มสามารถเลือกรับข้อมูลแบบ Full หรือ Basic ได้

Admin Approval

หากเปิดใช้งาน สัญญาณที่ EA ส่งมาจะมีสถานะ pending ก่อน ผู้ดูแลระบบต้องเข้าไปอนุมัติจึงจะส่ง LINE

ใช้เมื่อไหร่

เหมาะกับช่วงทดสอบหรือตลาดผันผวน ที่ต้องการตรวจสอบคุณภาพสัญญาณก่อนส่งไปยังสมาชิก

AI Analysis / Confidence

เมื่อเปิดใช้งาน ระบบจะเรียก OpenAI เพื่อประเมิน:

  • Risk:Reward ที่ดีที่สุดจาก Entry ถึง TP3 เทียบ SL
  • Confidence คะแนนความมั่นใจ 0-100
  • Summary สรุปจุดเด่น/จุดเสี่ยงของ setup

หากคะแนน confidence < minConfidence สัญญาณจะถูกปฏิเสธ

คำแนะนำ

เริ่มต้นที่ 60% แล้วค่อยปรับขึ้นหากสัญญาณดี หรือลดลงหากต้องการสัญญาณมากขึ้น

เป้าหมาย LINE

สามารถเพิ่มหลายเป้าหมายได้ แต่ละเป้าหมายประกอบด้วย:

  • ชื่อ — ใช้แสดงใน Log เท่านั้น
  • ประเภท — Group หรือ User
  • Plan — Full หรือ Basic
  • ID — LINE Group ID หรือ LINE User ID
  • เปิดใช้งาน — เปิด/ปิดการส่งของเป้าหมายนี้

จุดเข้า / ออก

จุดเข้า (Entry)

ราคาที่แนะนำให้เปิดออเดอร์ หากเปิด Multi-Entry จะมี 3 ระดับเพื่อกระจายจุดเข้า

  • SELL: Entry อยู่เหนือราคาปัจจุบัน (Sell Limit)
  • BUY: Entry อยู่ต่ำกว่าราคาปัจจุบัน (Buy Limit)

จุดออก (TP / SL)

  • TP1 / TP2 / TP3 — เป้าหมายทำกำไร แนะนำให้ปิดบางส่วนตามแต่ละ TP
  • SL — จุดตัดขาดทุน หากราคามาถึงระดับนี้ถือว่าสัญญาณผิด
กลยุทธ์แนะนำ

ใช้ Multi-Entry เพื่อลดความเสี่ยงจากการเข้าออเดอร์เพียงจุดเดียว หากราคาวิ่งผ่าน Entry 1 ไป ยังสามารถเข้าที่ Entry 2/3 ได้

โควต้า LINE / Logs

หน้า Admin จะแสดง:

  • โควต้าคงเหลือ — แพลนปัจจุบันและจำนวนข้อความที่ใช้ไป
  • สถิติการใช้งาน — จำนวนข้อความทั้งหมด / เดือนนี้ / การส่งซ้ำ
  • ประวัติการส่ง — รายการส่งล่าสุดพร้อมสถานะ success/failed
ระวัง

บัญชี LINE Messaging API แบบฟรีมีโควต้า 300 ข้อความ/เดือน ควรเปิด Multi-Entry เฉพาะเมื่อจำเป็น และตรวจสอบการส่งซ้ำจาก Log

แก้ไขปัญหาที่พบบ่อย

สัญญาณไม่ส่งไป LINE

  • ตรวจสอบว่าเปิด AI Analysis แล้ว Confidence ต่ำเกินไปหรือไม่
  • ตรวจสอบว่าเปิด Admin Approval แล้วลืมอนุมัติหรือไม่
  • ตรวจสอบว่ามีเป้าหมาย LINE ที่เปิดใช้งานและ ID ถูกต้อง
  • ตรวจสอบ LINE Token และโควต้า

Multi-Entry ราคาไม่ตรง

  • ตรวจสอบว่า EA ส่งค่า point มาหรือไม่ (ต้อง recompile EA หลังอัปเดต source)
  • ตรวจสอบว่าคู่เงินอยู่ในรายการที่ระบบรู้จำนวน Point size

ส่งซ้ำหลายครั้ง

  • ดูที่ส่วน ส่งซ้ำ ใน Admin
  • ตรวจสอบว่า EA ไม่ได้เปิดซ้ำบน chart เดียวกัน

คำถามที่พบบ่อย

Q: สามารถส่งไปหลายกลุ่มพร้อมกันได้ไหม?

A: ได้ ให้เพิ่มเป้าหมายหลายรายการในหน้า Admin แต่ละกลุ่มเลือก Plan ได้เอง

Q: เปลี่ยนระยะ Multi-Entry แล้วมีผลทันทีไหม?

A: มีผลทันทีกับสัญญาณถัดไปที่ EA ส่งมา สัญญาณเก่าไม่เปลี่ยน

Q: ปิด Multi-Entry แล้วจะกลับไปใช้ Entry เดียวไหม?

A: ใช่ ระบบจะกลับไปคำนวณ Entry/SL/TP จาก Order Block เหมือนเดิม

Q: ข้อความ LINE ใช้ AI เขียนหรือ Template?

A: ระบบจะลองใช้ AI เขียนก่อน หาก AI ล้มเหลวจะ fallback เป็นข้อความ Template อัตโนมัติ
