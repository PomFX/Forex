คู่มือระบบ EA Bridge

เอกสารฉบับสมบูรณ์สำหรับ BOS_LuxAlgo_Bridge.mq5 อธิบายตั้งแต่การติดตั้ง การตั้งค่า โฟลว์การทำงาน ไปจนถึงการอ่าน Log และแก้ไขปัญหาที่พบบ่อย

ภาพรวมระบบ

BOS_LuxAlgo_Bridge เป็น Expert Advisor (EA) บน MetaTrader 5 ที่ทำหน้าที่เป็นสะพาน (Bridge) ระหว่างอินดิเคเตอร์ LuxAlgo Smart Money Concepts (SMC) กับ Backend ของ ATH Trader

หน้าที่หลัก

อ่าน object BOS / CHoCH ที่ LuxAlgo วาดบน chart แล้วส่งข้อมูลสัญญาณไปยังเซิร์ฟเวอร์ เพื่อให้ AI วิเคราะห์ คิดระดับ TP/SL และส่งแจ้งเตือนไป LINE อัตโนมัติ

ข้อดี

ไม่ต้องเฝ้าหน้าจอตลอดเวลา ระบบช่วยกรองสัญญาณซ้ำ สัญญาณเก่า และสัญญาณที่ AI ไม่มั่นใจออกก่อนส่งไปกลุ่ม

EA นี้ทำงานร่วมกับระบบหลังบ้าน 2 ส่วนหลัก:

  • /api/signals/mt5/bos-candidate — รับข้อมูลสัญญาณจาก MT5
  • /api/ea/heartbeat — รับสถานะความเป็นอยู่ของ EA เป็นระยะ

องค์ประกอบของระบบ

ส่วนประกอบบทบาท
MetaTrader 5แพลตฟอร์มหลักที่รัน EA และแสดง indicator LuxAlgo SMC
LuxAlgo SMCวาด object BOS / CHoCH / Order Block บน chart ตาม Smart Money Concepts
BOS_LuxAlgo_Bridge.mq5EA ที่อ่าน object จาก chart แล้วส่งไป backend
Backend (Vercel)รับสัญญาณ เรียก AI วิเคราะห์ บันทึกฐานข้อมูล ส่ง LINE
LINE Messaging APIช่องทางส่งข้อความแจ้งเตือนไปยังกลุ่มหรือผู้ใช้
Admin Dashboardหน้าตั้งค่าการรับสัญญาณ กลุ่มเป้าหมาย approval และดูประวัติส่ง LINE

สิ่งที่ต้องเตรียม

  • บัญชี MetaTrader 5 (เวอร์ชันล่าสุดแนะนำ)
  • อินดิเคเตอร์ LuxAlgo Smart Money Concepts ติดตั้งและเปิดใช้งานบน chart
  • ไฟล์ EA BOS_LuxAlgo_Bridge.ex5 หรือ .mq5
  • API Key จากระบบ ATH Trader (ตั้งค่าใน Vercel env MT5_API_KEY)
  • URL backend ที่ถูกต้อง เช่น https://forex-rouge-gamma.vercel.app
  • อนุญาต WebRequest สำหรับ URL ดังกล่าวใน MT5

ติดตั้ง EA

1

คัดลอกไฟล์ EA

นำไฟล์ BOS_LuxAlgo_Bridge.ex5 ไปใส่ในโฟลเดอร์ MQL5/Experts ของ MT5 จากนั้นเปิด/รีสตาร์ท MT5 หรือคลิกขวาที่ Navigator เลือก Refresh

2

แนบ EA เข้า Chart

ลาก BOS_LuxAlgo_Bridge จาก Navigator มาวางบน chart ที่มี LuxAlgo SMC กำลังทำงานอยู่ แนะนำ Timeframe M15 หรือ M30 ตามความเหมาะสม

3

ยืนยัน AutoTrading

ตรวจสอบว่าปุ่ม AutoTrading บน toolbar ถูกเปิด (สีเขียว) EA นี้ไม่ได้เปิด order เอง แต่ต้องการสิทธิ์ให้ WebRequest ทำงานได้

4

ตั้งค่า Inputs

กรอก API_URL, API_KEY ให้ถูกต้อง ปรับ SEND_BOS, SEND_CHOCH, DEBOUNCE_BARS ตามต้องการ

5

อนุญาต WebRequest

ไปที่ Tools → Options → Expert Advisors แล้วเพิ่ม URL ของ backend ในรายการ Allowed WebRequest URLs

ตั้งค่า WebRequest ใน MT5

หากไม่เปิด WebRequest EA จะได้ error 4014 เมื่อเรียก API

  1. ใน MT5 ไปที่ Tools → Options
  2. เลือกแท็บ Expert Advisors
  3. ติ๊ก Allow WebRequest for listed URL
  4. เพิ่ม URL backend เช่น https://forex-rouge-gamma.vercel.app
  5. กด OK แล้วรีโหลด EA
เคล็ดลับ: บาง broker/VPS อาจต้องเปิด URL ทั้ง domain และ endpoint เช่น https://forex-rouge-gamma.vercel.app/api/signals/mt5/bos-candidate หาก domain อย่างเดียวยังติด

ตรวจจับสัญญาณ

EA ทำงานบน event OnTick() แต่จะประมวลผลจริงเฉพาะเมื่อมี แท่งใหม่ (new bar) เท่านั้น เพื่อลดภาระการคำนวณ

วิธีอ่าน Object จาก LuxAlgo

  1. สแกน object ทั้งหมดบน chart ด้วย ObjectsTotal()
  2. อ่านข้อความของ object จาก OBJPROP_TEXT
  3. ตรวจว่าข้อความมีคำว่า BOS หรือ CHoCH หรือไม่
  4. ข้าม object ที่เกี่ยวกับ EQL / Equilibrium (ไม่ใช่สัญญาณเทรด)
  5. อ่านเวลาและราคาของ object ที่พบ
  6. เลือก object ล่าสุดในแต่ละรอบ

ตรวจทิศทางจากสี

EA ใช้สีของ object จาก LuxAlgo เพื่อตัดสินใจว่าเป็นสัญญาณ bullish หรือ bearish

โฟลว์ Backend

หลังจาก EA ส่งข้อมูลมา ระบบหลังบ้านจะประมวลผลตามขั้นตอนดังนี้

1. รับ Candidate
→
2. บันทึก Signal
→
3. AI วิเคราะห์
→
4. ตรวจ Confidence
→
5. Approval
→
6. ส่ง LINE

รายละเอียดแต่ละขั้นตอน

  1. รับ Candidate — endpoint ตรวจสอบ X-MT5-Key และรับ JSON จาก EA
  2. บันทึก Signal — บันทึกข้อมูลลงฐานข้อมูลก่อนเสมอ แม้ AI จะปฏิเสธ
  3. AI วิเคราะห์ — เรียก OpenAI เพื่อประเมิน setup, คำนวณ TP/SL, R:R และให้คะแนนความมั่นใจ
  4. ตรวจ Confidence — หาคะแนนต่ำกว่า threshold ที่ตั้งไว้ ระบบจะปฏิเสธและไม่ส่ง LINE
  5. Approval — หากเปิดโหมด "ต้องอนุมัติก่อน" สัญญาณจะอยู่สถานะ pending จนกว่า Admin จะกดอนุมัติ
  6. ส่ง LINE — สัญญาณที่ผ่านทุกเงื่อนไขจะถูกส่งไปยังกลุ่มเป้าหมายตามการตั้งค่า
ข้อควรจำ

การบันทึก signal ก่อนเรียก AI ช่วยให้เราสามารถนำกลับมาวิเคราะห์ย้อนหลังได้ว่า AI ปฏิเสธอะไรบ้าง และปรับ prompt หรือ threshold ได้ถูกต้อง

ส่ง LINE ตามกลุ่มเป้าหมาย

ระบบรองรับการส่งข้อความไปยังหลายกลุ่ม LINE พร้อมกัน โดยแต่ละกลุ่มสามารถเลือกรูปแบบข้อความได้ว่าจะได้ข้อมูลเต็มหรือย่อ

Planข้อมูลที่ได้รับเหมาะกับ
Fullคู่เงิน ทิศทาง ราคาเข้า TP1-3 SL R:R เหตุผลVIP / กลุ่มเสียเงิน
Basicคู่เงิน ทิศทาง ราคาเข้า TP1 SL เท่านั้นกลุ่มฟรี / กลุ่มทดลอง

การตั้งค่ากลุ่มเป้าหมายทำได้ที่หน้า Admin MT5 Signal Settings โดยระบุ:

  • LINE Group ID หรือ User ID
  • Plan ที่ต้องการ (Full/Basic)
  • เปิด/ปิด การส่งของกลุ่มนั้น
โควต้า LINE: บัญชี LINE Messaging API แบบฟรีมีโควต้า 300 ข้อความ/เดือน ระบบจึงมีการกรองสัญญาณซ้ำและปิดการส่งสรุป BOS อัตโนมัติเพื่อประหยัดโควต้า

หน้า Admin MT5 Signal Settings

ผู้ดูแลระบบสามารถควบคุมพฤติกรรมของระบบสัญญาณได้ที่ /#/admin/mt5signals

ฟังก์ชันหลัก

  • เปิด/ปิด AI Analysis — ห้ปิด AI ระบบจะใช้ format เริ่มต้น
  • Confidence Threshold — ตั้งค่าคะแนนขั้นต่ำที่ AI ต้องได้จึงจะส่งสัญญาณ
  • Admin Approval — เปิดโหมดต้องอนุมัติก่อนส่ง LINE
  • Targets — เพิ่ม/ลบ/แก้ไขกลุ่ม LINE เป้าหมายและ Plan
  • Send Test — ทดสอบส่งข้อความไปกลุ่มทันที
  • Usage / Logs — ดูจำนวนข้อความที่ใช้ไปและประวัติการส่ง LINE

แก้ไขปัญหาที่พบบ่อย

Heartbeat HTTP 403

สาเหตุ: API Key ไม่ตรงกับ backend หรือ backend รับ key ผิดที่ ปัจจุบัน backend รองรับทั้ง X-MT5-Key header และ ?key=... แล้ว ตรวจสอบว่า API_KEY ใน EA ตรงกับ MT5_API_KEY บน Vercel

HTTP Error 4014

สาเหตุ: URL ไม่ได้อยู่ในรายการอนุญาตของ MT5 ให้ไปเพิ่ม https://forex-rouge-gamma.vercel.app ใน Tools → Options → Expert Advisors → Allowed WebRequest URLs แล้วรีโหลด EA

ไม่มีสัญญาณส่งไป LINE เลย

  • ตรวจสอบว่า LuxAlgo SMC วาด object BOS/CHoCH บน chart จริง
  • ตรวจสอบ log ว่ามี [Skip] Too old หรือ Debounce หรือไม่
  • ตรวจสอบ Confidence Threshold ใน Admin อาจสูงเกินไป
  • ตรวจสอบว่าเปิด Admin Approval แล้วลืมอนุมัติหรือไม่
  • ตรวจสอบ LINE Group ID / Token ถูกต้องและยังไม่หมดโควต้า

สัญญาณซ้ำหรือมากเกินไป

  • เพิ่มค่า DEBOUNCE_BARS ใน EA
  • ลดจำนวน timeframe ที่เปิด EA หลาย chart พร้อมกัน
  • ตรวจสอบว่าไม่ได้เปิด EA ซ้ำบน chart เดียวกัน

คำถามที่พบบ่อย

Q: เปิด EA หลาย chart พร้อมกันได้ไหม?

A: ได้ แต่ละ chart จะส่งตาม timeframe ของตัวเอง ควรตั้งชื่อ pair ให้ถูกต้องหรือปล่อยให้ auto-detect

Q: สามารถปิดการส่ง CHoCH แล้วส่งเฉพาะ BOS ได้ไหม?

A: ได้ ให้ตั้งค่า SEND_CHOCH = false และ SEND_BOS = true ใน Inputs

Q: หากไม่ต้องการ heartbeat ต้องทำอย่างไร?

A: ตั้งค่า HEARTBEAT_MIN = 0 แต่จะไม่สามารถดูสถานะ EA บน Dashboard ได้

Q: สัญญาณที่ AI ปฏิเสธจะถูกบันทึกไหม?

A: บันทึกในฐานข้อมูลเสมอ แต่จะไม่ส่ง LINE และสถานะอาจเป็น declined

Q: เปลี่ยน backend URL ต้องทำอะไรบ้าง?

A: แก้ไข API_URL ใน EA, อัปเดต MT5_API_KEY บนเซิร์ฟเวอร์ใหม่ และเพิ่ม URL ใหม่ใน MT5 Allowed WebRequest list
