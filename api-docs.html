ATH Trader API

REST API Documentation — สำหรับเชื่อมต่อ AI, Bot, หรือระบบภายนอก

Base URL: http://localhost:8080/api
Auth: ใช้ Bearer Token ใน Header Authorization: Bearer <token>

Auth

POST /api/auth/login NO AUTH
เข้าสู่ระบบ รับ JWT Token
FieldTypeRequiredDescription
emailstringyesอีเมลผู้ใช้
passwordstringyesรหัสผ่าน
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@athtrader.com","password":"admin123"}'

# Response: {"ok":true,"token":"eyJ...","user":{...}}
POST /api/auth/register NO AUTH
สมัครสมาชิกใหม่
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"user1","email":"user1@mail.com","password":"123456"}'

Signals (สัญญาณเทรด)

GET /api/signals NO AUTH
ดึงสัญญาณเทรดทั้งหมด (เรียงล่าสุดก่อน)
curl http://localhost:8080/api/signals

# Response: [{"id":1,"pair":"EUR/USD","direction":"BUY","entry":"1.08750",...}]
POST /api/signals ADMIN
เพิ่มสัญญาณเทรดใหม่
FieldTypeRequiredDescription
pairstringyesคู่เงิน (EUR/USD, XAU/USD, ...)
directionstringyesBUY หรือ SELL
entrystringnoราคาเข้า
tp1stringnoTake Profit 1
tp2stringnoTake Profit 2
tp3stringnoTake Profit 3
slstringnoStop Loss
statusstringnoactive / win / loss (default: active)
curl -X POST http://localhost:8080/api/signals \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"pair":"EUR/USD","direction":"BUY","entry":"1.08750","tp1":"1.09200","sl":"1.08400"}'
PUT /api/signals/:id ADMIN
แก้ไขสัญญาณเทรด (ส่งเฉพาะฟิลด์ที่ต้องการเปลี่ยน)
curl -X PUT http://localhost:8080/api/signals/1 \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"status":"win"}'
DELETE /api/signals/:id ADMIN
ลบสัญญาณเทรด
curl -X DELETE http://localhost:8080/api/signals/1 \
  -H "Authorization: Bearer <token>"

Articles (บทความ)

GET /api/articles NO AUTH
ดึงบทความทั้งหมด
POST /api/articles ADMIN
เพิ่มบทความใหม่
FieldTypeRequiredDescription
titlestringyesหัวข้อบทความ
contentstringyesเนื้อหาบทความ
imagestringnoURL รูปภาพประกอบ
curl -X POST http://localhost:8080/api/articles \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "วิเคราะห์ EUR/USD ประจำวัน",
    "content": "บทความวิเคราะห์โดย AI...",
    "image": "https://example.com/chart.jpg"
  }'
PUT /api/articles/:id ADMIN
แก้ไขบทความ
DELETE /api/articles/:id ADMIN
ลบบทความ

Brokers (โบรกเกอร์)

GET /api/brokers NO AUTH
ดึงรายชื่อโบรกเกอร์ทั้งหมด
POST /api/brokers ADMIN
เพิ่มโบรกเกอร์ใหม่
FieldTypeRequiredDescription
namestringyesชื่อโบรกเกอร์
descriptionstringnoคำอธิบาย
ibLinkstringnoIB Link URL
logostringnoURL โลโก้
ratingnumbernoคะแนน 1-5
PUT /api/brokers/:id ADMIN
แก้ไขโบรกเกอร์
DELETE /api/brokers/:id ADMIN
ลบโบรกเกอร์

Users (สมาชิก — Admin only)

GET /api/users ADMIN
ดึงรายชื่อสมาชิกทั้งหมด
PUT /api/users/:id/vip ADMIN
เปลี่ยนระดับ VIP ของสมาชิก
curl -X PUT http://localhost:8080/api/users/1/vip \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"vipLevel":"Gold"}'
DELETE /api/users/:id ADMIN
ลบสมาชิก

Stats (สถิติ)

GET /api/stats/public NO AUTH
สถิติหน้าแรก (totalSignals, buyWins, sellWins, vipCount)
GET /api/stats ADMIN
สถิติ Dashboard Admin

ตัวอย่าง: ให้ AI โพสต์บทความ

import requests

API = "http://localhost:8080/api"

# 1. Login ได้ token
res = requests.post(f"{API}/auth/login", json={
    "email": "admin@athtrader.com",
    "password": "admin123"
})
token = res.json()["token"]
headers = {"Authorization": f"Bearer {token}"}

# 2. AI เขียนบทความ
article = {
    "title": "วิเคราะห์ทองคำประจำวัน",
    "content": "ราคาทองคำวันนี้เคลื่อนไหวในกรอบ...",
    "image": "https://example.com/gold-chart.jpg"
}

# 3. โพสต์
r = requests.post(f"{API}/articles", json=article, headers=headers)
print(r.json())

# หรือโพสต์สัญญาณเทรด
signal = {
    "pair": "XAU/USD",
    "direction": "BUY",
    "entry": "2035.00",
    "tp1": "2042.00",
    "sl": "2028.00",
    "status": "active"
}
r = requests.post(f"{API}/signals", json=signal, headers=headers)
print(r.json())
Tips: Token ไม่หมดอายุ (เว้นแต่เปลี่ยน JWT_SECRET) สามารถเก็บไว้ใช้ยาวๆ หรือ login ใหม่ทุกครั้งก็ได้
